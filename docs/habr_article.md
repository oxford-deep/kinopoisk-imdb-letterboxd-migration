# Побег с Кинопоиска: переносим оценки в IMDb и Letterboxd и снова ищем людей с похожим вкусом

## Зачем всё это

У меня на Кинопоиске было около 1800 оценок.

Когда-то это было не просто «я поставил фильму циферку». Оценки реально работали как инструмент. На старом Кинопоиске можно было находить людей, чьи оценки совпадают с твоими на очень высокий процент.

Это была не социальная фича ради галочки, а почти антиспам-фильтр для кино:

```text
выходит фильм
↓
массовый рейтинг высокий
↓
смотришь оценки людей с похожим вкусом
↓
если они поставили низко — экономишь вечер
```

Проблема в том, что Кинопоиск перестал быть для меня удобным местом для такого сценария, а вся история оценок осталась там.

Задача была такая:

```text
вытащить оценки
↓
перенести их в IMDb
↓
перенести в Letterboxd
↓
найти людей с похожим вкусом
↓
использовать их оценки как фильтр
```

Звучит просто. На практике — не очень.

Все скрипты, примеры файлов и troubleshooting вынесены в репозиторий:

```text
https://github.com/<ваш-ник>/kinopoisk-imdb-letterboxd-migration
```

В статье оставил логику, команды и важные грабли.

## Что получилось

В моём случае:

```text
Кинопоиск: примерно 1820 оценок
CSV-экспорт: 1607 записей
автоматический матчинг с IMDb: 1338
дополнительный матчинг через alternative titles: +20
итого в IMDb: 1358 оценок
дальше импорт в Letterboxd
```

Главный вывод: перенос оценок — это только половина задачи. Настоящая цель — восстановить систему доверенных людей, чьи оценки помогают решать, смотреть фильм или нет.

## Экспорт из Кинопоиска

Первый очевидный путь — браузерные расширения. У меня одно из них упало с ошибкой:

```text
Can not find your user
```

Рабочим оказался старый Node/Puppeteer-экспортёр, но его пришлось патчить под текущую разметку Кинопоиска.

Старый скрипт ожидал элемент:

```js
select.navigator_per_page
```

На странице его уже не было.

Патч выглядел так:

```js
try {
  const el = await page.waitForSelector('select.navigator_per_page', { timeout: 5000 });
  await el.press('PageDown');
  await page.waitForNavigation({ timeout: 10000 }).catch(() => {});
} catch (e) {
  console.log('navigator_per_page not found, continuing...');
}
```

На выходе нужен CSV с колонками вроде:

```text
dateTime,url,isSeries,name,originalName,year,duration,isWatched,userVote,rating,votes,ratingImdb,votesImdb
```

Минимально важны:

```text
name
originalName
year
userVote
```

## Конвертация Кинопоиск → IMDb

IMDb не импортирует оценки по названию фильма. Нужен IMDb ID вида:

```text
tt0133093
```

Поэтому скрипт `kp_to_imdb.py` делает так:

```text
читает CSV Кинопоиска
↓
скачивает IMDb title.basics.tsv.gz
↓
ищет совпадения по названию и году
↓
создаёт imdb_import_ready.csv
```

Запуск:

```bash
python scripts/kp_to_imdb.py ratings.csv
```

Выходные файлы:

```text
imdb_import_ready.csv
matched_review.csv
unmatched.csv
```

Первый проход дал 1338 совпадений из 1607 записей.

## Попытка добить остатки через IMDb AKAs

Для альтернативных названий есть `title.akas.tsv.gz`.

Запуск:

```bash
python scripts/kp_to_imdb_akas.py unmatched.csv
```

Идея хорошая: локальные фильмы часто лежат в IMDb под другим названием.

На практике это тяжёлый шаг. В моём случае он дал только +20 совпадений. Поэтому я бы относился к нему как к необязательной добивке, а не как к обязательной части маршрута.

## Импорт в IMDb

Официального нормального CSV-импорта оценок в IMDb я не нашёл.

Но при ручном выставлении оценки IMDb отправляет GraphQL mutation:

```graphql
mutation UpdateTitleRating($rating: Int!, $titleId: ID!) {
  rateTitle(input: {rating: $rating, titleId: $titleId}) {
    rating {
      value
    }
  }
}
```

Для импорта я использовал JS из репозитория:

```text
scripts/imdb_bulk_rater_console.js
```

Сценарий:

```text
открыть IMDb
залогиниться
открыть DevTools Console
вставить JS
выбрать imdb_import_ready.csv
дождаться завершения
```

Результат: 1338 оценок импортировались без ошибок. Потом отдельно импортировались ещё 20.

## IMDb → Letterboxd

Дальше всё проще:

```text
IMDb → Your Ratings → Export
Letterboxd → Settings → Import & Export → Import your data
```

Letterboxd переводит шкалу 1–10 в 0.5–5 звёзд:

```text
IMDb 10 → 5★
IMDb 9  → 4.5★
IMDb 8  → 4★
...
```

После этого профиль Letterboxd уже не пустой, а содержит историю оценок.

## Что не сработало для поиска похожих людей

Я пробовал идти техническим путём:

```text
compare-сервисы
самописный Python
Puppeteer
парсинг профилей Letterboxd
```

На практике это быстро превращается в маленький Кинопоиск на коленке:

```text
403
security check
таймауты
неполные данные
много шума
```

Отдельные инструменты для Letterboxd могут дать стартовый список похожих пользователей. Это полезно, но не волшебная кнопка. В моей выборке более-менее похожими оказались примерно 2 человека из 10.

## Что реально сработало

Самый рабочий способ оказался проще.

Берём фильмы-маркеры.

Не фильмы, которые любят все, а именно те, которые хорошо разделяют людей:

```text
спорный фильм, который вы любите
фильм, который многие хвалят, а вы не любите
фильм, где ваша оценка сильно отличается от средней
```

Дальше:

```text
открываем фильм в Letterboxd
↓
идём в рецензии
↓
фильтруем по своей оценке
↓
открываем профили людей, которые поставили так же
↓
в профиле оставляем только общие фильмы
↓
быстро смотрим совпадения и расхождения
↓
если человек похож — Follow
```

Почему рецензии удобны? Их часто пишут люди, у которых уже много оценок. Если у человека 1000+ оценённых фильмов, он полезен как ориентир: у него будет много того, что вы ещё не смотрели.

## Как пользоваться дальше

Когда вы подписались на 5–10 похожих людей, не надо бегать по каждому профилю.

Открываете страницу фильма и смотрите:

```text
Activity from friends
Reviews from people you follow
Watched by friends
Liked by friends
```

И это решает исходную задачу:

```text
думаю смотреть фильм
↓
открываю Letterboxd
↓
смотрю оценки людей с похожим вкусом
↓
принимаю решение
```

Плюс Letterboxd начинает сам показывать активность друзей: кто что недавно посмотрел и как оценил. Это превращается в живую рекомендательную ленту, где вместо абстрактного алгоритма — люди, которых вы сами отобрали.

## Что делать с новыми оценками

Нормальной двусторонней синхронизации IMDb ↔ Letterboxd нет.

Варианты:

```text
IMDb главный → периодически экспортировать в Letterboxd
Letterboxd главный → IMDb оставить архивом
оценивать в двух местах → путь сильных духом
```

Для моей задачи логичнее сделать Letterboxd основным сервисом для новых оценок, потому что социальная часть живёт именно там.

Если когда-нибудь понадобится перегнать новые оценки обратно в IMDb, Letterboxd умеет экспортировать данные, а IMDb можно обновить тем же GraphQL-скриптом.

## Итог

Если у вас много оценок на Кинопоиске, маршрут такой:

```text
Кинопоиск
↓
CSV
↓
IMDb ID
↓
IMDb
↓
Letterboxd
↓
фильмы-маркеры
↓
Follow
↓
Activity from friends
```

Не надо идеально переносить 100% оценок.

Если из 1800 оценок качественно переехали 1300–1500, этого уже достаточно, чтобы восстановить профиль вкуса.

Главное — не забыть, зачем всё это делается. Не ради красивой таблички, а ради простого бытового решения:

```text
смотреть это сегодня вечером или не надо портить себе жизнь
```
