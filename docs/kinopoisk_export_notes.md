# Экспорт оценок из Кинопоиска

Этот шаг нужен, чтобы получить файл:

```text
ratings.csv
```

Дальше именно этот CSV используется для конвертации в IMDb и импорта в Letterboxd.

В репозитории есть готовый Puppeteer-экспортёр:

```text
scripts/kinopoisk_votes_exporter.js
```

Он открывает страницу оценок Кинопоиска в браузере, проходит по страницам `/votes/` и сохраняет оценки в CSV.

---

## Установка

Нужен Node.js.

В корне проекта выполните:

```bash
npm install 
```

Puppeteer может скачать совместимый Chromium/Chrome. Это нормально.

---

## Запуск

Лучше использовать числовой ID профиля Кинопоиска.

```bash
node scripts/kinopoisk_votes_exporter.js "https://www.kinopoisk.ru/user/<numeric_id>/votes/" ratings.csv
```

Пример формата URL:

```text
https://www.kinopoisk.ru/user/123456/votes/
```

Числовой ID профиля обычно надёжнее, чем URL с ником.

Вместо:

```text
https://www.kinopoisk.ru/user/some_nickname/votes/
```

лучше пробовать:

```text
https://www.kinopoisk.ru/user/<numeric_id>/votes/
```

---

## Как работает экспортёр

Скрипт:

```text
открывает браузер
переходит на страницу votes
даёт пройти логин / капчу вручную, если нужно
собирает ссылки на фильмы и сериалы
пытается вытащить название, оригинальное название, год и вашу оценку
проходит по страницам
сохраняет ratings.csv после каждой страницы
```

CSV сохраняется в UTF-8 with BOM, чтобы Excel меньше ломал русские названия.

---

## Что должно получиться

На выходе нужен CSV с оценками.

Желательный набор колонок:

```text
dateTime,url,isSeries,name,originalName,year,duration,isWatched,userVote,rating,votes,ratingImdb,votesImdb
```

Минимально важные колонки:

```text
name
originalName
year
userVote
```

Где:

```text
name          локальное / русское название
originalName  оригинальное название
year          год фильма
userVote      ваша оценка от 1 до 10
```

Обезличенный пример лежит здесь:

```text
examples/kinopoisk_ratings_example.csv
```

## Что сработало в исходной миграции

Сначала браузерное расширение для экспорта упало с ошибкой:

```text
Can not find your user
```

После этого сработал путь через Node/Puppeteer.

Старый экспортёр пришлось переписать и сделать устойчивее, потому что он ждал элемент:

```js
select.navigator_per_page
```

На текущей странице его могло не быть.

В текущем `scripts/kinopoisk_votes_exporter.js` этот кейс уже обработан:

```js
try {
  const $navigatorPerPage = await page.waitForSelector('select.navigator_per_page', { timeout: 5000 });
  await $navigatorPerPage.press('PageDown');
  await page.waitForNavigation({ timeout: 10000 }).catch(() => {});
} catch {
  console.log('navigator_per_page not found, continuing...');
}
```

Также убраны опасные обязательные `waitForNavigation()`, которые могли зависнуть, если страница никуда не переходила.

## Если Кинопоиск покажет логин или капчу

Скрипт запускает браузер в видимом режиме.

Кинопоиск может показать:

```text
логин
капчу
проверку безопасности
страницу с ограничением доступа
```

Что делать:

```text
1. пройти проверку руками в открытом браузере
2. вернуться в терминал
3. нажать Enter
```

После этого экспорт продолжится.

---

## Почему экспорт может дать меньше оценок, чем есть в профиле

Даже успешный экспорт может дать меньше записей, чем показывает профиль.

Причины:

```text
часть оценок относится к сериалам
часть относится к ТВ-шоу
страницы имеют разную разметку
Кинопоиск поменял интерфейс
часть данных не попала в текущий votes list
экспортёр не смог разобрать часть карточек
```

Для дальнейшей задачи это не всегда критично.

Цель — не музейная точность архива, а перенос основного ядра оценок.

Если из условных 1800 оценок удалось качественно выгрузить 1300–1600, это уже рабочая база для Letterboxd.

## Что проверить перед конвертацией

Перед запуском:

```bash
python scripts/kp_to_imdb.py ratings.csv
```

откройте `ratings.csv` и проверьте:

```text
есть ли колонка userVote
есть ли год фильма
есть ли originalName или name
оценки находятся в диапазоне 1–10
файл сохранён в UTF-8
```

Если Excel показывает кракозябры, открывайте CSV через импорт:

```text
Excel
→ Data
→ From Text/CSV
→ File Origin: 65001 UTF-8
→ Delimiter: comma
```

Или используйте Google Sheets.

## Минимальный тестовый запуск

Можно быстро проверить экспортёр на своём профиле:

```bash
npm install puppeteer
node scripts/kinopoisk_votes_exporter.js "https://www.kinopoisk.ru/user/<numeric_id>/votes/" ratings.csv
```

После первых страниц уже должен появиться файл:

```text
ratings.csv
```

Скрипт сохраняет файл по ходу работы, а не только в самом конце.

## Если `userVote` пустой

Без `userVote` дальнейший импорт в IMDb невозможен, потому что это ваша оценка.

Проверьте:

```text
вы открыли именно свой профиль / votes
вы залогинены
на странице реально видны ваши оценки
Кинопоиск не показывает публичную страницу без оценок
```

Если оценки на странице видны, но `userVote` в CSV пустой, значит Кинопоиск поменял разметку блока оценки.

Тогда нужно обновлять селекторы в файле:

```text
scripts/kinopoisk_votes_exporter.js
```

В первую очередь смотрите функцию:

```js
parseUserVote()
```

## Главное

Экспорт с Кинопоиска возможен.

На момент этой миграции рабочий путь выглядел так:

```text
Puppeteer exporter
↓
числовой ID профиля
↓
ручной проход логина / капчи при необходимости
↓
ratings.csv
```

После получения CSV дальше маршрут уже стабильнее:

```text
ratings.csv
↓
IMDb ID
↓
IMDb
↓
Letterboxd
```
